    • ReactiveSocket

    Application protocol with Reactive Streams semantics.

    Project maintained by NetflixOSS



    ReactiveSocket is an application protocol providing Reactive Streams semantics over an asynchronous, binary boundary.

    It enables the following symmetric interaction models via async message passing over a single connection:

    • request/response (stream of 1)
    • request/stream (finite stream of many)
    • fire-and-forget (no response)
    • event subscription (infinite stream of many)
    • channel (bi-directional streams)

    Information can be found in these documents:

    • Motivations.md - Why ReactiveSocket?
    • FAQ.md - Frequently Asked Questions
    • Protocol.md - The Protocol

    ReactiveSocket is for communicating across network boundaries thus it is intended to be polyglot:

    • reactivesocket-java
    • reactivesocket-js
    • reactivesocket-cpp

    The core project for Java implements the protocol and exposes Reactive Stream APIs. Typically most use will come via another library that uses the core. For example:

    • ReactiveSocket over TCP using Netty reactivesocket-transport-tcp
    • ReactiveSocket over WebSockets using Netty reactivesocket-transport-websocket
    • ReactiveSocket over Aeron reactivesocket-transport-aeron

    If you would like to contribute an implementation in another language or for different libraries, platforms or frameworks of your choice, please contact us to explore creating a new repo.
